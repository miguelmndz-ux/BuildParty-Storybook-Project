Token Naming Audit

5 issues found. Each one has a reason and a suggested fix. Multi-select any you want to keep as-is.

Issue 1 · Tier 1 — Primitive
radius-6 is named after its value
Tier 1
radius-6
→
radius-sm
Same problem as font-size-14 — if you ever change the value to 8px, the name becomes a lie. T-shirt naming (radius-sm) keeps the name stable across value changes and is consistent with your font-size convention.
Issue 2 · Tier 2 — Semantic
color-action-text is ambiguous
Tier 2
color-action-text
→
color-on-action
"action-text" reads as "the text color for actions" — but this token is actually the white text that sits on top of the amber button. The "on-" convention (from Material Design) is now standard: color-on-action unambiguously means "color used on top of the action surface."
Issue 3 · Tier 2 — Semantic
color-action-border describes structure, not purpose
Tier 2
color-action-border
→
color-action-emphasis
This token gets used as both the outlined button's border and its text color — so calling it "border" is already inaccurate. The semantic meaning is "the strong, accent-weight expression of the action color." color-action-emphasis describes that intent without locking in a structural assumption.
Issue 4 · Tier 2 — Semantic
size-button-* in the semantic tier leaks component knowledge
Tier 2
size-button-padding-y
size-button-padding-x
size-button-radius
size-button-font-size
size-button-font-weight
→
size-control-padding-y
size-control-padding-x
size-control-radius
size-control-font-size
size-control-font-weight
Semantic tokens should be component-agnostic — they describe purpose, not the component that consumes them. "Control" is the standard design system term for interactive UI elements (buttons, inputs, selects). When you later add an Input or Select component, they'll naturally share size-control-* tokens instead of each needing their own duplicate set.
Issue 5 · Convention — Minor
font-size-sm and font-weight-semibold mix two naming conventions
Tier 1
font-size-sm
font-weight-semibold
→
font-size-sm ✓
font-weight-semibold ✓
sm is a scale position. semibold is a typographic descriptor. Both are widely accepted — this is a heads up, not a hard fix. The alternative would be font-weight-3 (numeric scale) but that's less readable. These names are fine — just be consistent if you add more tokens later.
